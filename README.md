# autotest — Python Reliability Shield

Maps PRD requirements to tests, detects weak assertions via mutation testing, and produces a reliability score. Works on any Python project without the need for external services like databases.

## Features

- **Requirement Traceability**: Parses Markdown PRDs and links `REQ-ID`s directly to pytest test cases.
- **Mutation Bridge**: Correlates survivors generated by `mutmut` directly back to the `coverage.py` context that *should* have killed them, revealing weak test assertions.
- **Reliability CLI**: Analyzes results to classify every requirement as `VERIFIED`, `WEAK`, `FAILING`, or `UNTESTED`, producing an aggregate reliability score.
- **100% Local**: No Neo4j, no databases. Everything operates on JSON and local files.

## Install

```bash
pip install -e .
```

## Usage

### 1. Tag your PRD

```markdown
# Product Requirements
## REQ-101: User must register with a valid email
## REQ-102: System rejects users under 18
```

### 2. Tag your tests

Add the marker to your `pytest.ini` or `pyproject.toml`:

```toml
[tool.pytest.ini_options]
markers = ["requirement: REQ-ID this test validates"]
```

Apply the markers in your test suite:

```python
import pytest

@pytest.mark.requirement("REQ-101")
def test_registration_valid_email():
    assert True
```

### 3. Register the Traceability Plugin

In your `conftest.py`, ensure the plugin tracks test outcomes:

```python
from autotest.conftest_plugin import TraceabilityPlugin
import pytest

@pytest.fixture(autouse=True, scope="session")
def _traceability_plugin(request):
    plugin = TraceabilityPlugin(store_path="traceability_store.json")
    request.config.pluginmanager.register(plugin)
```

### 4. Run the Pipeline

```bash
# 1. Generate coverage + traceability data (requires pytest-cov)
pytest --cov=src --cov-context=test --cov-report=json

# 2. Run mutation testing (mutmut 3.x)
mutmut run

# 3. Bridge the mutation survivors to your tests
python -m autotest.mutation_bridge \
  --meta-dir mutants \
  --coverage coverage.json \
  --store traceability_store.json \
  --output mutation_results.json

# 4. Generate the reliability report
python -m autotest.report --prd prd.md --store traceability_store.json --threshold 90
```

## How It Works

1. `conftest_plugin.py`: Intercepts `pytest` logs. Every test tagged with `@pytest.mark.requirement("REQ-X")` writes its execution duration, outcome, and markers to `traceability_store.json`.
2. `mutmut`: Mutates code and runs tests. It drops JSON `.meta` files into a `mutants/` directory.
3. `mutation_bridge.py`: Parses the `.meta` files and `coverage.json`. If a mutant survived on a line of code, the bridge identifies which test covered that file and flags the test as "weak".
4. `report.py`: Compiles the final matrix.

## Custom Mutation Configuration

Copy `autotest/templates/mutmut_config.py` to your project root to exclude logging strings or tweak mutation boundaries.
